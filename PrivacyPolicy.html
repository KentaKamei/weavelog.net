Weavelog プライバシーポリシー

トップ / プライバシーポリシー

1. はじめに

本プライバシーポリシーは、「Weavelog」（以下「当アプリ」といいます。）が収集する利用者情報とその取り扱いについて説明するものです。当アプリの利用者（以下「ユーザー」といいます。）は、本ポリシーの内容をご確認・ご理解の上、当アプリをご利用ください。

2. 取得する情報の種類

当アプリは、以下の利用者情報を取得します。

A) ユーザーから直接提供いただく情報

  • メールアドレス: ユーザー認証、アカウント管理、および重要なお知らせの連絡のために使用します。
  • ユーザー名: アプリ内でユーザーを識別し、他のユーザーに表示するために使用します。
  • プロフィール写真: 任意で設定された場合、プロフィール画面に表示するために使用します。
  • パスワード: Firebase Authentication のシステム内で安全に管理され、当アプリ運営者が直接閲覧することはありません。
  • 内部ユーザー識別子（UID）: ユーザーアカウントを識別し、投稿、フォロー関係、通知、設定などのすべてのユーザー関連データを紐付けるために使用される一意の識別子。

B) ユーザーが投稿する情報

  • 投稿のタイトル・本文: 日記投稿の内容として使用します。
  • 添付画像: 任意で投稿に添付された画像ファイル。
  • ハッシュタグ: 投稿の分類・検索機能のために使用します。
  • 公開範囲設定: 非公開・フォロワーのみ・全体公開の選択情報。
  • リアクション情報: 他のユーザーの投稿への「がんばれ！」などのリアクションの記録。
  • 投稿テンプレートの内容: 投稿時に利用するテンプレートとして保存されたテキスト内容、ハッシュタグなど。

C) ソーシャル機能で収集する情報

  • フォロー関係: ユーザー間のフォロー・フォロワー情報。
  • フォローリクエスト: プライベートアカウントへのフォローリクエスト情報。
  • ブロック関係: ユーザー間のブロック情報。
  • 通報情報: ユーザーからの不適切なコンテンツや行動の通報。
  • 通知設定: プッシュ通知の受信設定。

D) サービスの利用により自動的に収集する情報

当アプリは、サービスの品質向上のため、以下の情報を自動的に収集することがあります。

  • 利用統計情報: 投稿数、フォロワー数、継続日数（ストリーク）、バッジの取得状況など、ユーザーの活動に関する統計データ。
  • デバイス情報: OS の種類とバージョン、デバイスのモデル名、アプリバージョン、IP アドレスなど。
  • クラッシュ情報: アプリが予期せず終了した場合に、その原因を調査するための技術的なログ。
  • 管理者操作ログ: アプリの管理者が実行した操作（お知らせの作成、問い合わせへの返信、ユーザーの BAN 等）に関する記録（操作日時、管理者 ID、操作内容）。
  • プッシュ通知トークン: プッシュ通知送信のために端末固有の FCM トークンを自動収集し、トークン更新時に自動的に更新されます。
  • バッジの取得記録: 特定の活動（例: 初投稿、継続日数達成など）に応じて付与されるバッジの取得履歴。
  • 広告 ID: 広告をパーソナライズして表示するために、iOS の IDFA または Android の AAID を取得します。
  • 広告視聴履歴: リワード広告の視聴回数、視聴日時、取得した特典（投稿枚数増加など）の記録。

E) デバイスアクセス情報

  • カメラへのアクセス: ユーザーが投稿やプロフィール写真のために写真を撮影できるよう、カメラへのアクセスを要求します。撮影された画像のみがアプリ内で利用されます。

F) お問い合わせに関する情報

  • お問い合わせ内容: ユーザーが入力した件名、詳細、カテゴリ、添付ファイル。
  • 運営からの返信内容: 問い合わせに対する運営からの回答。
  • 対応状況: 「未対応」「対応中」などのステータス。
  • （お問い合わせ時に自動収集する）端末情報: OS の種類、アプリバージョン、端末モデルなど、問題解決のために利用する技術情報。
※ 当アプリが収集しない情報
当アプリは以下の情報は一切収集いたしません：
  • 位置情報（GPS 情報）
  • 連絡先情報
  • 他のアプリの利用履歴
  • 端末内の写真・ファイル（投稿時に選択されたもの以外）

3. 利用目的

当アプリは、取得した情報を以下の目的で利用します。

  1. アカウント管理: ユーザー認証、アカウント情報の管理、およびサービスの円滑な提供のため。
  2. 投稿・交流機能: ユーザーの投稿を表示し、他のユーザーとの交流（フォロー・リアクション）を可能にするため。
  3. 通知機能: 以下の目的でプッシュ通知を送信するため：
    • 投稿継続のリマインダー通知
    • フォロー・フォローリクエスト通知
    • 投稿へのリアクション通知
    • 運営からの重要なお知らせ
  4. 継続サポート機能: ストリーク計算、バッジ付与、習慣トラッキングなど、ユーザーの継続をサポートする機能を提供するため。
  5. 検索・発見機能: ハッシュタグやユーザー名による検索、新しい投稿の発見機能を提供するため。
  6. ブロック機能: 特定のユーザーからのコンテンツ表示を制限するため。
  7. 通報対応: アプリの安全性を維持し、利用規約違反に対処するため。
  8. 管理者によるユーザー管理: アプリの健全な運営と利用規約違反への対処のため（BAN など）。
  9. 管理者操作の監査: アプリの健全な運営とセキュリティ維持のため、管理者の操作を記録し監査するため。
  10. お知らせ機能: ユーザーへのお知らせや重要な情報を伝達するため。
  11. データ管理とアクセス制御: ユーザーのデータ（投稿、設定など）を正確に管理し、適切なアクセス権限を適用するため。
  12. 問い合わせ対応: ユーザーからのお問い合わせを管理し、返信や問題解決を行うため。
  13. サービス改善: サービスの利用状況を分析し、サービスの改善や新機能の開発に役立てるため。
  14. 技術的保守: アプリ内で発生した不具合（クラッシュ）の原因を調査し、修正するため。
  15. プッシュ通知送信: 重要なお知らせやアプリ内イベント（フォロー通知、リアクション通知、リマインダーなど）をリアルタイムで配信するため。
  16. 広告配信: ユーザーの興味や行動履歴に基づいてパーソナライズされた広告を表示するため（AdMob を利用）。
  17. 投稿制限機能: リワード広告の視聴による投稿枚数の増加特典を管理するため。

4. 第三者への提供について

当アプリは、以下の場合を除き、ユーザーの同意なく個人情報を第三者に提供することはありません。

  • 法令に基づき開示が求められた場合。
  • 人の生命、身体または財産の保護のために必要がある場合。

また、当アプリは、以下の目的でサードパーティ製のツールを利用しており、それに伴い利用者情報が各ツール提供事業者に送信されることがあります。

事業者名: Google LLC

利用するサービス:

  • Firebase: アプリのバックエンド機能（認証、データベース、プッシュ通知、クラッシュレポート等）の提供のため。
  • Firebase Cloud Messaging (FCM): プッシュ通知の送信サービス。端末固有トークンを使用して通知を配信します。
  • Firebase Analytics: サービスの利用状況を分析し、サービス改善に役立てるため。
  • AdMob: パーソナライズド広告の配信サービス。ユーザーの興味関心に基づいた広告を表示し、リワード広告による投稿枚数増加特典を提供します。

送信される情報: 上記「2-D) サービスの利用により自動的に収集する情報」に記載の情報。

利用目的: 上記「3. 利用目的」に記載の目的。

各サービスのプライバシーポリシー:

  • Google プライバシーポリシー
  • Google 広告に関するポリシー

5. ユーザーの権利について

  • 投稿の削除: ユーザーは、アプリ内の操作により、いつでも自身の投稿を削除することができます。削除された投稿は即座にサーバーから除去されます。
  • プロフィール情報の変更: ユーザー名、プロフィール写真は、アプリ内の設定画面からいつでも変更可能です。
  • 通知設定の変更: プッシュ通知の受信設定は、アプリ内の設定画面からいつでも変更可能です。
  • アカウントの削除: アカウント全体の削除を希望される場合は、アプリ内のお問い合わせ機能より、その旨をご連絡ください。本人確認の上、以下の手順で削除手続きを行います：
    • ユーザーアカウント情報の削除
    • 投稿データの完全削除
    • フォロー関係の削除
    • 関連する通知データの削除

※ 削除されたデータは復旧することができません。ただし、法令上の義務や不正利用の調査のため、または個人を特定できないように匿名化された統計データについては、アカウント削除後も一定期間保持することがあります。

  • お問い合わせ内容
  • 必要に応じてスクリーンショット等

7. プライバシーポリシーの変更

当アプリは、法令の変更やサービスの改善に伴い、本プライバシーポリシーを改定することがあります。重要な変更があった場合は、アプリ内での通知など、分かりやすい方法でお知らせします。
